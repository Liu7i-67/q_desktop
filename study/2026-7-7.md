# 如何在browser-window-created事件中对webContents的will-navigate、select-bluetooth-device、media-paused等操作进行拦截，如何进行测试拦截是否生效

在 `browser-window-created` 中集中管控 `webContents` 是大厂标准的安全实践。

这里需要先纠正一个概念：**`will-navigate` 和 `select-bluetooth-device` 是可以被“拦截（阻止行为）”的，但 `media-paused` 只是一个纯粹的“状态通知”事件，无法被拦截或取消**，你只能监听它来做状态同步。

以下是主进程拦截代码以及对应的渲染进程测试方案。

### 一、 主进程：集中管控代码

在你的 `main.js` 或 `WindowManager` 中，监听窗口创建并挂载拦截逻辑：

```javascript
import { app, shell } from 'electron';

app.on('browser-window-created', (event, win) => {
  const { webContents } = win;

  // 1. 拦截页面跳转 (will-navigate)
  webContents.on('will-navigate', (event, url) => {
    // 允许本地开发地址或特定的内网地址
    if (
      url.startsWith('http://127.0.0.1') ||
      url.startsWith('https://your-domain.com')
    ) {
      return; // 正常放行
    }

    // 拦截其他所有外部跳转
    event.preventDefault();
    console.log(`[安全管控] 已拦截非法路由跳转: ${url}`);

    // 可选方案：将拦截下的合法外链交由用户的默认浏览器（如 Chrome/Edge）打开
    // shell.openExternal(url);
  });

  // 2. 拦截蓝牙设备选择 (select-bluetooth-device)
  // 当网页调用 navigator.bluetooth.requestDevice 时触发
  webContents.on('select-bluetooth-device', (event, deviceList, callback) => {
    event.preventDefault(); // 阻止 Electron 弹出默认的丑陋设备选择框

    console.log(
      '[安全管控] 检测到蓝牙请求，设备列表:',
      deviceList.map((d) => d.deviceName),
    );

    // 策略 A：直接拒绝请求
    callback('');

    // 策略 B：如果有白名单，自动静默选择特定设备
    // const targetDevice = deviceList.find(d => d.deviceName === 'My-Device');
    // if (targetDevice) callback(targetDevice.deviceId);
    // else callback('');
  });

  // 3. 监听媒体暂停 (media-paused)
  // 注意：这是被动通知，没有 event.preventDefault()。当网页中的 <video> 或 <audio> 暂停时触发。
  webContents.on('media-paused', () => {
    console.log('[状态同步] 检测到媒体暂停');
    // 典型应用：在这里清除任务栏的“正在播放”角标，或暂停主进程的某些高频轮询任务
  });

  // 补充：媒体开始播放的监听
  webContents.on('media-started-playing', () => {
    console.log('[状态同步] 检测到媒体开始播放');
  });
});
```

### 二、 渲染进程：测试触发代码

为了测试上述拦截是否生效，你可以在前端（React/Vue 或纯 HTML）写三个触发器：

```javascript
// 在你的前端页面（例如 App.tsx）中放置以下触发函数

// 1. 测试 will-navigate
const testNavigate = () => {
  // 尝试改变顶层 window 的地址。如果是 SPA 内部路由(history.push)不会触发，必须是真实的 URL 跳转
  window.location.href = 'https://www.google.com';
};

// 2. 测试 select-bluetooth-device
const testBluetooth = async () => {
  try {
    // 请求蓝牙设备，这会触发主进程的 select-bluetooth-device
    const device = await navigator.bluetooth.requestDevice({
      acceptAllDevices: true,
    });
    console.log('获取设备成功:', device);
  } catch (error) {
    // 如果主进程执行了 callback('')，这里会捕获到 NotFoundError 异常
    console.error('蓝牙获取被拒绝或取消:', error);
  }
};

// 3. 测试 media 状态
const testMedia = () => {
  // 动态创建一个音频元素并播放/暂停
  const audio = new Audio(
    'https://interactive-examples.mdn.mozilla.net/media/cc0-audio/t-rex-roar.mp3',
  );

  audio.play().then(() => {
    console.log('音频播放中...');
    // 1秒后自动暂停，此时主进程应该打印出 [状态同步] 检测到媒体暂停
    setTimeout(() => {
      audio.pause();
    }, 1000);
  });
};
```

**测试效果验证：**

1. 点击跳转按钮，页面**不应该**跳转，主进程控制台打印拦截日志。
2. 点击蓝牙按钮，前端控制台会打印 `NotFoundError` 报错，且屏幕上**没有任何弹窗提示**。
3. 点击播放测试，主进程会依次打印出媒体播放和暂停的日志。
